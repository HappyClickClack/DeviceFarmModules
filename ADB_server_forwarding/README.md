## Проброс общего ADB сервера в контейнеры с Jenkins узлами
<img width="1226" height="766" alt="image" src="https://github.com/user-attachments/assets/e284a334-4e3b-4293-9f5f-2445fb7074f5" />

Для работы с ADB устройствами из docker контейнеров традиционно используется проброс портов или устройств в контейнер и запуск ADB сервера в контейнере, об этих решениях есть множество информации в интернете, не будем останавливаться на этих способах. Здесь же рассмотрим еще один вариант - это проброс ADB сервера в контейнер. Преимуществом такого подхода является общий ADB сервер, который может быть использован в разных контейнерах одновременно, например, в контейнерах с разными Jenkins узлами, а также в контейнерах с web интерфейсом, health-check-ом и т.п. Также стоит отметить недостатки этого подхода: общий ADB сервер является точкой отказа всей схемы и при его падении пропадают устройства во всех контейнерах. Дополнительной особенностью такого решения является доступность всех устройств во всех контейнерах, это может быть как недостатком, например, если по ошибке Jenkins узел одного устройства может обратиться к другому устройству, так и достоинством, например, если это health-check контейнер и ему нужно видеть все устройства.

Для реализации такой схемы нужно выполнить два шага:
- шаг первый - запустить общий ADB сервер с привязкой ко всем интерфейсам. Вот команда для запуска в терминале Linux `adb -a start-server`. Если нужен режим `nodaemon`, то вот так `adb -a nodaemon server start &> /dev/null &`
 <br>_, здесь `-a` — bind ко всем интерфейсам (0.0.0.0), а не только localhost, чтобы подключаться к ADB извне (по сети/из контейнера/с другой машины), `&> /dev/null` — перенаправляет и stdout, и stderr в /dev/null (то есть “ничего не выводить”, включая ошибки), `&` — запускает команду в фоне._
- шаг второй - передать в контейнеры для работы с устройствами вот такие дополнительные параметры:
```
docker run -d \
 --add-host=host.docker.internal:host-gateway \
 -e "ANDROID_ADB_SERVER_ADDRESS=host.docker.internal" \
 -e "ANDROID_SERIAL=1234567890" \
...

```
 <br>_, здесь `--add-host=host.docker.internal:host-gateway` — добавляет IP адрес хоста в /etc/hosts внутри контейнера, `-e "ANDROID_ADB_SERVER_ADDRESS=host.docker.internal"` — определяет переменную окружения с указанием хостового ADB сервера для работы, `ANDROID_SERIAL` - спецификатор устройства._

После этого можно проверять доступность устройств внутри контейнеров, для этого выполните внутри контейнера `adb devices -l`. 
Стоит отметить, что версии ADB везде должны быть совместимы или совпадать.

Если не использовать `-a` при старте ADB сервера на хосте, то при использовании ADB в контейнере будет ошибка `network.cpp:149 failed to connect to 'host.docker.internal:5037': Connection refused`.

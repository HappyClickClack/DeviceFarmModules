## Проброс общего ADB сервера в контейнеры с Jenkins узлами
<img width="866" height="538" alt="image" src="https://github.com/user-attachments/assets/5776ddd1-2923-495a-a0ed-c94891bdaf8d" />

Для работы с ADB устройствами из docker контейнеров традиционно используется проброс портов или устройств в контейнер и запуск ADB сервера в контейнере, об этих решениях есть множество информации в интернете, не будем останавливаться на этих способах. Здесь же рассмотрим еще один вариант - это проброс ADB сервера в контейнер. Преимуществом такого подхода является общий ADB сервер, который может быть использован в разных контейнерах одновременно, например, в контейнерах с разными Jenkins узлами, а также в контейнерах с web интерфейсом, health-check-ом и т.п. Также стоит отметить недостатки этого подхода: общий ADB сервер является точкой отказа всей схемы и при его падении пропадают устройтсва во всех контейнерах. Дополнительной особенностью такого решения является достпуность всех устройств во всех контейнерах, это может быть как недостатком, например, если по ишибке Jenkins узел одного устройства может обратиться к другому устройству, так и достоинством, например, если это health-check контейнер и ему нужно видеть все устройства.

Для реализации такой схемы нужно выполнить два шага:
- шаг первый - запустить общий ADB сервер  режиме `nodaemon` чтобы он мог независимо работать с каждым контейнером. Вот команда для запуска в терминале Linux `adb -a nodaemon server start &> /dev/null &`
 <br>_, здесь `-a` — bind ко всем интерфейсам (0.0.0.0), а не только localhost, чтобы подключаться к ADB извне (по сети/из контейнера/с другой машины), `&> /dev/null` — перенаправляет и stdout, и stderr в /dev/null (то есть “ничего не выводить”, включая ошибки), `&` — запускает команду в фоне._
- шаг второй - передать в контейнеры для работы с устройствами вот такие дополнительные параметры:
```
docker run -d \
 --add-host=host.docker.internal:host-gateway \
 -e "ANDROID_ADB_SERVER_ADDRESS=host.docker.internal" \
 -e "ANDROID_SERIAL=1234567890" \
...

```
 <br>_, здесь `--add-host=host.docker.internal:host-gateway` — добавляет IP адрес хоста в /etc/hosts внутри контейнера, `-e "ANDROID_ADB_SERVER_ADDRESS=host.docker.internal"` — определяет переменную окружения с указанием хостового ADB сервера для работы, `ANDROID_SERIAL` - спецификатор устройства._

После этого можно проверять доступнсть устройств внутри контейнеров, для этого выполните внутри контейнера `adb devices -l`. 
Стоит отметить, что версии всех используемых ADB должны быть совместимы или совпадать.
